# Customer Lifecycle Optimisation — Index

> Pillar 4 of the Monetization Strategy Knowledge Base. Covers how to move customers through the revenue lifecycle — from potential to converted, from converted to expanded, from at-risk to retained, from churned to resurrected.

---

## The Optimisation-First Principle

Before changing the monetization model (pricing, packaging, value metric), exhaust optimisation strategies. Model changes are high-risk and slow to test. Optimisation changes are lower-risk and faster to validate.

```
Default sequence:
  1. Optimise customer transitions within the current model
  2. If optimisation is exhausted → consider model changes (see foundation/02-monetization-model-structure.md)
```

## Customer States

Every customer exists in one of four states at any given time. The right strategy depends on accurately identifying which state a customer is in.

| State | Definition | Primary Goal |
|---|---|---|
| **Potential** | Has not yet paid; in or before the conversion funnel | Convert |
| **Existing Healthy** | Paying and engaged; showing positive signals | Expand |
| **At-Risk** | Paying but showing disengagement or churn signals | Retain |
| **Churned** | Was paying; no longer paying | Resurrect (selectively) |

See `customer-lifecycle/01-customer-states.md` for full diagnosis criteria.

## The Universal Transition Model

All customer movement follows the same underlying logic:

```
Educate → Convert → Activate
```

This applies at every stage:
- **Potential customers:** Educate (on the problem/solution) → Convert (first payment) → Activate (Aha Moment)
- **Existing customers expanding:** Educate (on upgrade value) → Convert (to higher tier) → Activate (on new features)
- **At-risk customers:** Educate (re-engage with value) → Convert (from at-risk to healthy) → Activate (re-establish habit)
- **Churned customers:** Educate (re-introduce product improvements) → Convert (re-subscribe) → Activate (re-establish habit)

## The Conversion and Retention Equations

```
Customer converts when:
  Perceived Value of Target State > Perceived Price + Friction to Convert

Customer retains when:
  Perceived Value of Staying > Perceived Value of Churning + (Effort to Stay − Friction of Leaving)
```

Every strategy in this section is designed to move one or more variables in these equations in the customer's favour.
