# Promo Codes

> Part of the Discount Mechanisms series. See [discount-mechanisms.md](../discount-mechanisms.md) for the full index.

---

## What It Is

Promo codes (also called discount codes, coupon codes, or voucher codes) are unique alphanumeric strings distributed through marketing channels, partnerships, or campaigns, redeemable at checkout for a defined discount. The code is the delivery vehicle for the discount — what makes promo codes distinctive is their flexibility: the same underlying discount mechanism (first month free, 30% off, etc.) can be distributed through multiple channels simultaneously, each with a unique code for attribution.

The defining characteristic is **controlled distribution and measurable attribution**. Unlike an on-site discount that is visible to everyone, a promo code goes only to the audience the distribution channel reaches. This makes promo codes the instrument of choice when the discount strategy requires channel-specific targeting, influencer or partnership distribution, or performance measurement against specific acquisition sources.

Promo codes work on the same psychological principles as other discounts (reduced price barrier, perceived value) but add a layer of **exclusivity and identity**: a user who has a code feels like they have inside access — they were chosen, they are getting something the general public does not have.

## Variants

### Influencer / Creator Codes
A unique code distributed by a creator or influencer to their audience. The creator typically receives an affiliate commission per conversion (separate from the discount itself). The code functions as both a tracking mechanism for the partnership and a discount for the audience.

*Example: "TECHREVIEWER20" for 20% off. Every conversion using this code is attributable to the creator's channel.*

### Partnership / Co-Marketing Codes
Codes distributed through a partner company's channels — an app store bundle, a newsletter partnership, a bank or e-wallet cashback campaign, or a hardware bundle inclusion.

*Example: A productivity app included in a university student pack, each student receiving a unique code for a free 3 months.*

### Campaign Attribution Codes
Internal codes created for specific marketing campaigns — email, paid social, direct mail. Each channel gets a unique code, enabling conversion attribution at the channel level.

*Example: EMAILJAN26, FBJAN26, IGJAN26 — three codes, same 20% discount, different channels to measure which drove the most conversions.*

### Event / Conference Codes
One-time codes distributed at industry events, product demos, or networking contexts. Often short-window validity (used before or during an event).

### Support Goodwill Codes
Codes generated by the support team and issued to users experiencing issues, as a gesture of goodwill. Not a marketing mechanic — a CX tool.

### Multi-Use vs. Single-Use Codes
- **Multi-use codes** (a single code redeemable unlimited times) are simple to distribute but cannot prevent leakage — the code spreads beyond the intended audience.
- **Single-use codes** (each code valid once) require a larger code generation and distribution infrastructure but prevent exploitation and enable per-redemption tracking.

## Performance in B2C

Promo codes are a highly reliable conversion tool with well-understood performance characteristics, but they carry the highest leakage risk of any discount mechanism.

**Documented performance:**
- Promo code conversion rates from influencer/creator audiences typically run 5–15% of code-sharing audience — materially better than standard ad conversion rates, with lower CPM/CPA due to the creator's built-in trust.
- Partnership codes in student or professional bundles show strong cohort quality — users acquired through a structured partnership (university, professional association) tend to show higher LTV than users acquired through general advertising.
- Campaign attribution codes enable real-time CAC measurement per channel, making them a finance and growth team favourite for budget allocation decisions.

**Leakage risk:**
Multi-use promo codes leak. Once a code appears on a coupon aggregator site (RetailMeNot, Honey, browser extensions that auto-apply coupons), it will be used by audiences far beyond the intended distribution. For acquisition campaigns where any new subscriber is welcome, this is a feature. For targeted campaigns where the discount is intended for a specific cohort (e.g., a student programme), leakage to general audiences delivers discounted subscriptions to users who were never the intended beneficiary.

**Stacking prevention:** Many platforms allow users to enter multiple promo codes. If the system doesn't prevent stacking, users may combine a promo code with an annual plan discount and a referral credit, resulting in near-zero first-period revenue. Stacking rules must be explicit in the billing system.

**SEA context:** Promo codes work well in SEA, particularly through influencer and creator channels on platforms like TikTok, Instagram, and YouTube. Partner codes distributed through e-wallets (e.g., "First subscription via GoPay with code X gets cashback") are particularly effective because they leverage the e-wallet's cashback culture.

**EU compliance:** Promo codes that display a "reference price" alongside the discounted price are subject to EU Omnibus Directive requirements. A 30% off code must be applied to a reference price that was the actual price for the prior 30 days.

## Best For

- Influencer and creator partnership programmes where per-code attribution and commission tracking is required
- B2B partnership distributions (bundled with partner products, student programmes, professional associations)
- Multi-channel campaigns where attribution at the channel level is a priority
- Support and CX goodwill gestures for individual users
- Events and conferences where a one-time contextual offer is appropriate

**Not ideal for:**
- Acquisition campaigns where the audience is unlimited — an on-site discount or flash sale is simpler and equally effective without the code infrastructure
- Products with billing systems that don't support promo code management well — operational complexity is real
- Situations where leakage would be damaging (exclusive programme codes that become publicly available)

## Revenue Model Fit

| Revenue Model | Fit | Notes |
|---|---|---|
| [Freemium](../models/01-freemium.md) | High | Promo codes for first month of paid tier free or discounted; or codes that grant temporary premium access to free users (trial via code). |
| [Free Trial (Traditional)](../models/02-free-trial.md) | Medium | Extended trial codes ("30 days free instead of 14 with this code") are a clean, low-risk promo code application. |
| [Reverse Trial](../models/03-reverse-trial.md) | Medium | Partner or influencer codes that trigger a reverse trial on signup (rather than standard freemium) can work for targeted acquisition campaigns. |
| [Flat-Rate Subscription](../models/04-flat-rate-subscription.md) | Very High | Simple percentage or fixed-amount discount on a single subscription price — cleanest promo code implementation. |
| [Tiered Subscription](../models/05-tiered-subscription.md) | Very High | Tier-specific codes (code unlocks Pro tier for X months free), or cross-tier upgrade codes. Very flexible. |
| [One-Time Purchase](../models/06-one-time-purchase.md) | Very High | Promo codes are the native instrument for one-time purchase discounts. Universally supported, simple to implement. |
| [Usage-Based](../models/07-usage-based.md) | High | Codes that grant bonus credits on first purchase ("Code CREATOR25 adds 25% bonus credits to your first pack") are a clean usage-model promo mechanic. |
| [Hybrid Models](../models/08-hybrid.md) | High | Flexible: subscription discount codes, credit bonus codes, or feature unlock codes can each be deployed independently across the hybrid model's components. |

## UX Flow

### Phase 1 — Code Distribution (External)

**Goal:** Ensure codes reach the right audience with the right context.

```
Channel distribution:
   → Influencer / creator: Code shared in video description, post caption, or verbal mention
   → Partner: Code distributed in partner's onboarding email, app, or bundle inclusion
   → Campaign: Code in email, paid ad, or landing page targeting specific cohort
   → Event: Code on printed material, slide deck, or event registration confirmation
        ↓
Code format best practices:
   → Memorable, readable codes (CREATOR20, not TG7X9KL2) for verbally shared codes
   → Random alphanumeric for single-use codes (not guessable)
   → Clear expiry communicated at distribution point
```

---

### Phase 2 — Code Entry at Checkout

**Goal:** The code entry experience must be low-friction and non-intrusive. Users without a code must not feel like they're missing something.

```
Pricing / checkout page
        ↓
[Code entry placement — two acceptable patterns]:

Option A: Visible at plan selection
   → "Have a promo code?" link (collapsed by default — not a prominent empty box)
   → Expands on click: text field + "Apply" button
   → Visible but not attention-grabbing to users without a code

Option B: On payment screen only
   → Promo code field appears at payment details step
   → Later in flow; only users who know they have a code will look for it
        ↓
User enters code:
   [Valid code]
   → Immediate feedback: "Code applied — X% off your first month"
   → Price updates in real-time in the summary
   → No additional confirmation required

   [Invalid / expired code]
   → Clear error: "This code is invalid or has expired"
   → Suggest: "Check the original source for the correct code"
   → Standard pricing remains; no friction to proceed without code

   [Already redeemed — single use]
   → "This code has already been used"
   → Standard pricing; no indication of fraud — just move forward
```

---

### Phase 3 — Code-Applied Checkout Completion

**Goal:** Confirm the discount clearly and set honest expectations about the post-code pricing.

```
Payment confirmation screen
   → "Promo code [CODE] applied — [X]% off your first month"
   → First billing amount confirmed
   → Standard renewal amount and date clearly stated
   → CTA into product
        ↓
Post-purchase email
   → "Your [Product] subscription is active — promo code [CODE] applied"
   → Receipt showing discounted first amount and standard renewal amount
```

---

### Phase 4 — Attribution & Reporting (Backend)

**Goal:** Ensure the code attribution data reaches the growth and finance teams for decision-making.

```
Each code redemption logged:
   → Code used
   → Acquisition channel (if code is channel-specific)
   → User ID
   → Conversion amount (discounted + projected renewal)
   → Timestamp
        ↓
Reporting dashboard (growth team)
   → Conversions per code
   → Revenue per code (accounting for discount)
   → LTV tracking at 30/60/90 days per code cohort
   → Leakage detection: codes used outside expected distribution channel
```
